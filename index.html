다음 중 LEFT OUTER JOIN에 대한 설명으로 올바른 것은?
① INNER JOIN은 왼쪽 테이블의 모든 행을 반환한다
② LEFT OUTER JOIN은 오른쪽 테이블에 일치하는 행이 없으면 NULL로 채운다
③ LEFT OUTER JOIN은 INNER JOIN보다 항상 결과 행 수가 적다
④ 두 JOIN의 결과는 항상 동일하다
✅ 정답
② LEFT OUTER JOIN은 오른쪽 테이블에 일치하는 행이 없으면 NULL로 채운다
LEFT JOIN은 왼쪽 테이블 행을 전부 유지하고, 오른쪽에 매칭이 없으면 NULL로 채움. 그래서 결과 행 수는 INNER JOIN ≤ LEFT JOIN.

SELECT e.name, d.dept_name

✅ 정답

개념 Q22
LEFT JOIN과 WHERE절을 조합했을 때 의도치 않게 INNER JOIN처럼 동작하는 경우를 예시 쿼리와 함께 설명하고, 올바른 쿼리도 함께 작성하라.
✅ 정답
-- ❌ 잘못된 예 (INNER JOIN처럼 동작)
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d ON e.dept_no = d.dept_no
WHERE d.dept_name = '개발팀';
-- WHERE절에서 NULL인 행이 필터링 → LEFT JOIN 효과 사라짐
 
-- ✅ 올바른 예 (ON절에 조건 이동)
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d ON e.dept_no = d.dept_no
    AND d.dept_name = '개발팀';
WHERE는 JOIN 후 필터링 → NULL 행 제거. ON은 JOIN 시 조건 적용 → NULL 행 유지. LEFT JOIN의 "모두 포함" 효과를 살리려면 조건을 ON절에 써야 함.
개념 Q23
CROSS JOIN은 언제 사용하는가? 이 예제의 테이블을 활용해 실용적인 사용 예를 설명하라.
✅ 정답
ON 조건 없이 두 테이블의 모든 행 조합(카테시안 곱)을 생성할 때 사용.

이 예제에서의 활용: products(3개) × colors(3개) = 9가지 상품-색상 조합표 생성. 두 테이블 사이에 FK 관계가 없어도 가능한 모든 경우의 수를 나열할 때 적합.
다른 사용 예: 날짜 테이블 × 시간 슬롯 테이블 → 모든 예약 가능 슬롯 생성.
개념 Q24
ON 조건에서 NULL = NULL이 FALSE인 이유를 설명하고, 이것이 이 예제의 JOIN 결과에 어떤 영향을 미치는지 설명하라.
✅ 정답
SQL에서 NULL은 "알 수 없는 값"이므로, NULL = NULL의 결과는 TRUE가 아니라 UNKNOWN으로 처리됨. WHERE/ON 조건에서 UNKNOWN은 FALSE와 동일하게 작동.

이 예제의 영향: 최유리(dept_no = NULL)는 ON e.dept_no = d.dept_no 조건이 절대 성립하지 않으므로, INNER JOIN 시 항상 제외됨. LEFT JOIN을 사용해도 왼쪽 행(최유리)은 유지되지만 오른쪽(departments)은 NULL로 채워짐.
NULL 비교는 반드시 IS NULL / IS NOT NULL을 사용해야 함. WHERE dept_no = NULL은 항상 0행 반환.
개념 Q25
INNER JOIN / LEFT OUTER JOIN / RIGHT OUTER JOIN / CROSS JOIN / SELF JOIN / 비등치 조인의 차이를 각각 한 줄로 설명하고, 이 예제에서 어떤 테이블 조합에 해당하는지 써라.
✅ 정답
JOIN 종류 설명 이 예제
INNER 양쪽 모두 매칭되는 행만 반환 employees ↔ departments
LEFT OUTER 왼쪽 전부 + 오른쪽 없으면 NULL 부서 없는 직원 포함
RIGHT OUTER 오른쪽 전부 + 왼쪽 없으면 NULL 직원 없는 부서(영업팀) 포함
CROSS ON 없이 모든 행 조합 (카테시안 곱) products × colors = 9행
SELF 같은 테이블끼리 조인 (별칭 2개 사용) employees 직원 ↔ 매니저
비등치 = 대신 BETWEEN/범위 조건 사용 salary ↔ salary_grade 등급
